An easy to use ProgressDialog library for Android API level 24 and above. The below given parameters apply for all the overloaded constructors in Java or a Single Constructor with default arguments in Kotlin.

Parameters

context

The Activity context which must be provided for sure no matter which overloaded constructor is called.

modeConstant

The Int constant, which is an optional parameter that accepts either MODE_DETERMINATE or MODE_INDETERMINATE. If it is not passed, MODE_INDETERMINATE will be set by default, which can be changed later by setting mode.

themeConstant

The Int constant, which is an optional parameter that accepts either THEME_DARK, THEME_LIGHT, or THEME_FOLLOW_SYSTEM If it is not passed, THEME_LIGHT will be set by default, which can be changed later by setting theme.

NOTE : THEME_FOLLOW_SYSTEM can be used starting from Android API Level 30 (Android 11) only. Attempting to use it in lower versions will throw IllegalArgumentException.

setOnShowListener

@JvmOverloads
fun setOnShowListener(isTimeTrackingEnabled: Boolean = false, onShowListener: DialogInterface.OnShowListener?)
Content copied to clipboard

Sets the DialogInterface.OnShowListener for ProgressDialog. Time Tracking is disabled by Default. It can be enabled by passing true to the isTimeTrackingEnabled parameter of the function. The onshowListener parameter can be null when isTimeTrackingEnabled parameter is passed. If null is passed for onShowListener when isTimeTrackingEnabled is not passed, no listener will be set. Enabling Time Tracking will make isCancelable to true automatically, since Time Tracking is an indefinite operation. It is recommended to set NEGATIVE BUTTON using setNegativeButton Method or set DialogInterface.OnDismissListener using setOnDismissListener after enabling Time Tracking feature. Time Tracking will stop when progress reaches maxValue or when the Dialog is dismissed or cancelled.
